# FIRE Calculator — Design Philosophy

**Plain and simple: we design for the couple planning their future, not for a spreadsheet.**

---

- **One hero answer per view.** FIRE age and nest-egg number come first — before any slider. The user should know "when can we quit?" in one glance, then drill into levers.

- **Chunk the complexity.** Four numbered sections (Target, Home, Household, Expenses) break a intimidating financial model into digestible pieces. Collapse a section when you're done — the summary line keeps context without clutter.

- **Progress feels like a game.** Positive green when savings rate is strong. Milestone language in copy ("flywheel at $500k", "Coast-FIRE territory"). Friendly greetings, not corporate dashboards.

- **Jargon behind tooltips.** Every slider and stat has a plain label upfront. The "i" icon holds the full explanation — withdrawal rates, CPP/OAS bridge, marginal tax — for when you're ready to learn, not when you're just checking the number.

- **Big targets, few decisions.** Sliders are large and easy to grab. One primary action per card. No nested menus or hidden settings on the main plan screen.

- **Airy, not empty.** Generous padding (32px screen, 40px between sections) gives financial data room to breathe. Brutalist borders provide structure without cramming content.

- **Honest, not scary.** Disclaimers stay small and below the hero — visible but never blocking the exciting answer. The app is a rough model for exploration, not financial advice.

- **Consistent, not clever.** Same card style, same button style, same header pattern everywhere. Sheets, assistant, and export all feel like the same app.

- **Friendly, not frivolous.** The tone is warm and a bit playful ("when can you ditch the 9-to-5?") but the numbers are real. Fun copy lowers the anxiety of confronting retirement math.

---

*Summary: Hero answer first. Chunk complexity. Gamify progress. Jargon in tooltips. Big sliders. Lots of air. Honest disclaimers. Brutalist structure, human voice.*
